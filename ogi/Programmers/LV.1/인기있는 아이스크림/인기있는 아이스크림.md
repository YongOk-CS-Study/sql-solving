# SQL 문법 정리

## 인기있는 아이스크림

### 사용한 SQL

```sql
SELECT FLAVOR
FROM FIRST_HALF
ORDER BY TOTAL_ORDER DESC, SHIPMENT_ID;
```

### 핵심 문법

#### 1. 특정 컬럼 조회

```sql
SELECT FLAVOR
```

`SELECT`는 조회할 컬럼을 선택할 때 사용한다.

이 문제에서는 아이스크림의 맛을 출력해야 하므로 `FLAVOR` 컬럼만 선택한다.

#### 2. FROM

```sql
FROM FIRST_HALF
```

`FROM`은 데이터를 조회할 테이블을 지정할 때 사용한다.

이 문제에서는 상반기 아이스크림 총주문량 정보가 들어 있는 `FIRST_HALF` 테이블에서 데이터를 조회한다.

#### 3. ORDER BY 내림차순 정렬

```sql
ORDER BY TOTAL_ORDER DESC
```

`ORDER BY`는 조회 결과를 특정 컬럼 기준으로 정렬할 때 사용한다.

`DESC`는 내림차순 정렬을 의미한다.

이 문제에서는 총주문량이 많은 순서대로 아이스크림 맛을 출력해야 하므로 `TOTAL_ORDER DESC`를 사용한다.

#### 4. ORDER BY 다중 정렬

```sql
ORDER BY TOTAL_ORDER DESC, SHIPMENT_ID
```

정렬 조건은 여러 개를 사용할 수 있다.

앞에 있는 조건이 먼저 적용되고, 해당 값이 같은 행끼리 다음 조건으로 다시 정렬된다.

- `TOTAL_ORDER DESC`: 총주문량이 많은 순서대로 정렬
- `SHIPMENT_ID`: 총주문량이 같다면 출하 번호가 작은 순서대로 정렬

`ASC`는 오름차순 정렬을 의미하며 기본값이다.

따라서 `SHIPMENT_ID`는 아래처럼 작성해도 같은 의미이다.

```sql
ORDER BY TOTAL_ORDER DESC, SHIPMENT_ID ASC
```

### 추가로 알아둘 내용

#### 1. ORDER BY의 기본 정렬 방향

`ORDER BY`에서 정렬 방향을 생략하면 기본적으로 오름차순으로 정렬된다.

```sql
ORDER BY SHIPMENT_ID
```

위 문장은 아래 문장과 같다.

```sql
ORDER BY SHIPMENT_ID ASC
```

이 문제의 정답 SQL에서 `SHIPMENT_ID` 뒤에 `ASC`를 생략할 수 있는 이유도 이 때문이다.

#### 2. 정렬 조건의 우선순위

```sql
ORDER BY TOTAL_ORDER DESC, SHIPMENT_ID ASC
```

정렬 조건이 여러 개 있을 때는 왼쪽부터 차례대로 우선순위가 적용된다.

예를 들어 아래와 같은 데이터가 있다고 하자.

| FLAVOR | TOTAL_ORDER | SHIPMENT_ID |
| ------ | ----------- | ----------- |
| melon  | 3000        | 3           |
| choco  | 3000        | 1           |
| vanilla | 4000       | 2           |

정렬 결과는 다음과 같다.

| FLAVOR | TOTAL_ORDER | SHIPMENT_ID |
| ------ | ----------- | ----------- |
| vanilla | 4000       | 2           |
| choco  | 3000        | 1           |
| melon  | 3000        | 3           |

먼저 `TOTAL_ORDER DESC`에 의해 총주문량이 많은 `vanilla`가 가장 위에 온다.

그다음 총주문량이 같은 `choco`와 `melon`은 `SHIPMENT_ID ASC` 조건에 따라 출하 번호가 작은 `choco`가 먼저 온다.

#### 3. SELECT에는 출력할 컬럼만 작성한다

문제에서 요구하는 출력 컬럼은 `FLAVOR`뿐이다.

따라서 정렬 기준으로 사용하는 `TOTAL_ORDER`, `SHIPMENT_ID`를 반드시 `SELECT`에 포함할 필요는 없다.

```sql
SELECT FLAVOR
FROM FIRST_HALF
ORDER BY TOTAL_ORDER DESC, SHIPMENT_ID ASC;
```

`ORDER BY`에서는 `SELECT`에 없는 컬럼도 정렬 기준으로 사용할 수 있다.

#### 4. ASC와 DESC의 차이

```sql
ORDER BY TOTAL_ORDER ASC
```

`ASC`는 오름차순이므로 숫자가 작은 값부터 큰 값 순서로 정렬된다.

```sql
ORDER BY TOTAL_ORDER DESC
```

`DESC`는 내림차순이므로 숫자가 큰 값부터 작은 값 순서로 정렬된다.

이 문제는 "총주문량이 큰 순서"를 요구하므로 `DESC`를 사용해야 한다.
