# SQL 문법 정리

## 잔챙이 잡은 수 구하기

### 사용한 SQL

```sql
SELECT COUNT(*) AS FISH_COUNT
FROM FISH_INFO
WHERE LENGTH IS NULL;
```

### 문제 이해하기

이 문제는 잡은 물고기 정보가 들어 있는 `FISH_INFO` 테이블에서 길이가 `10cm` 이하인 물고기의 수를 구하는 문제입니다.

문제 조건에서 `10cm` 이하의 물고기는 `LENGTH` 컬럼에 실제 길이 값이 아니라 `NULL`로 저장되어 있습니다.

따라서 `LENGTH`가 `NULL`인 행만 찾으면 잔챙이 물고기의 수를 구할 수 있습니다.

출력 컬럼명은 문제에서 요구한 대로 `FISH_COUNT`로 지정합니다.

### 핵심 문법

#### 1. COUNT(*)

```sql
COUNT(*)
```

`COUNT(*)`는 조건에 맞는 행의 개수를 세는 집계 함수입니다.

이 문제에서는 잔챙이에 해당하는 물고기가 몇 마리인지 세야 하므로 `COUNT(*)`를 사용합니다.

#### 2. AS

```sql
AS FISH_COUNT
```

`AS`는 출력되는 컬럼에 별칭을 붙일 때 사용합니다.

문제에서 결과 컬럼명을 `FISH_COUNT`로 출력하라고 했기 때문에 `COUNT(*)`의 결과에 `FISH_COUNT`라는 별칭을 붙입니다.

#### 3. WHERE

```sql
WHERE LENGTH IS NULL
```

`WHERE`는 조회할 행을 조건으로 필터링할 때 사용합니다.

이 문제에서는 길이가 `10cm` 이하인 물고기만 세야 하므로, `LENGTH` 값이 `NULL`인 행만 조회합니다.

#### 4. IS NULL

```sql
LENGTH IS NULL
```

SQL에서 `NULL`은 값이 없음을 의미합니다.

`NULL`은 일반적인 값처럼 `=` 연산자로 비교할 수 없기 때문에, `IS NULL`을 사용해서 확인해야 합니다.

```sql
WHERE LENGTH = NULL
```

위와 같은 방식은 올바른 조건식이 아닙니다.

`NULL` 여부를 확인할 때는 반드시 다음처럼 작성합니다.

```sql
WHERE LENGTH IS NULL
```

### 풀이 흐름

1. `FISH_INFO` 테이블에서 데이터를 조회합니다.
2. `LENGTH IS NULL` 조건으로 잔챙이 물고기만 필터링합니다.
3. `COUNT(*)`로 필터링된 행의 개수를 셉니다.
4. 결과 컬럼명을 `FISH_COUNT`로 출력합니다.

### 최종 정리

이 문제의 핵심은 `10cm` 이하의 물고기가 `NULL`로 저장된다는 조건을 파악하는 것입니다.

그래서 숫자 비교 조건을 쓰는 대신 `LENGTH IS NULL` 조건으로 행을 찾고, `COUNT(*)`로 개수를 세면 됩니다.

### 다른 풀이법

#### 1. SUM과 CASE WHEN 사용하기

```sql
SELECT SUM(
    CASE
        WHEN LENGTH IS NULL THEN 1
        ELSE 0
    END
) AS FISH_COUNT
FROM FISH_INFO;
```

`CASE WHEN`은 조건에 따라 다른 값을 반환할 때 사용합니다.

이 풀이에서는 `LENGTH`가 `NULL`이면 `1`, 아니면 `0`으로 바꾼 뒤 `SUM()`으로 모두 더합니다.

즉, 잔챙이 물고기는 `1`로 계산되고, 잔챙이가 아닌 물고기는 `0`으로 계산됩니다.

예를 들어 데이터가 다음과 같다고 하면,

| LENGTH |
| --- |
| NULL |
| 25 |
| NULL |
| 30 |

`CASE WHEN` 결과는 다음처럼 바뀝니다.

| LENGTH | CASE 결과 |
| --- | --- |
| NULL | 1 |
| 25 | 0 |
| NULL | 1 |
| 30 | 0 |

따라서 `SUM()` 결과는 `1 + 0 + 1 + 0 = 2`가 됩니다.

#### 2. 전체 개수에서 LENGTH가 있는 행의 개수 빼기

```sql
SELECT COUNT(*) - COUNT(LENGTH) AS FISH_COUNT
FROM FISH_INFO;
```

`COUNT(*)`는 전체 행의 개수를 셉니다.

반면 `COUNT(LENGTH)`는 `LENGTH`가 `NULL`이 아닌 행만 셉니다.

따라서 전체 물고기 수에서 길이 값이 있는 물고기 수를 빼면, `LENGTH`가 `NULL`인 물고기 수를 구할 수 있습니다.

예를 들어 전체 행이 `5개`이고, `LENGTH` 값이 있는 행이 `3개`라면 다음과 같이 계산됩니다.

```text
COUNT(*) - COUNT(LENGTH)
= 5 - 3
= 2
```

이 방식도 정답을 구할 수 있지만, `NULL`인 행을 직접 찾는 풀이보다 의도가 조금 덜 직관적일 수 있습니다.

#### 3. 원래 풀이와 비교

```sql
SELECT COUNT(*) AS FISH_COUNT
FROM FISH_INFO
WHERE LENGTH IS NULL;
```

가장 추천하는 풀이는 위와 같은 원래 풀이입니다.

문제에서 요구하는 조건이 `LENGTH가 NULL인 물고기의 수`이기 때문에, `WHERE LENGTH IS NULL`로 조건을 직접 표현하는 것이 가장 읽기 쉽습니다.

`SUM(CASE WHEN ...)` 방식은 조건별 개수를 여러 개 구해야 할 때 유용합니다.

`COUNT(*) - COUNT(LENGTH)` 방식은 `COUNT(컬럼)`이 `NULL`을 제외하고 센다는 특징을 이해하는 데 도움이 됩니다.
