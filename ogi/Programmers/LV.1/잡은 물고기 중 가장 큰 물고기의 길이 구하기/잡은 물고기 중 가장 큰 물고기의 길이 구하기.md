# SQL 문법 정리

## 잡은 물고기 중 가장 큰 물고기의 길이 구하기

### 사용한 SQL

```sql
SELECT CONCAT(FORMAT(MAX(LENGTH), 2), 'cm') AS MAX_LENGTH
FROM FISH_INFO;
```

### 핵심 문법

#### 1. MAX 함수

```sql
MAX(LENGTH)
```

`MAX()`는 특정 컬럼에서 가장 큰 값을 구하는 집계 함수이다.

이 문제에서는 잡은 물고기 중 가장 긴 길이를 구해야 하므로 `LENGTH` 컬럼에 `MAX()`를 사용한다.

```sql
SELECT MAX(LENGTH)
FROM FISH_INFO;
```

위처럼 작성하면 가장 큰 길이 값만 조회할 수 있다.

#### 2. MAX만 사용했을 때 소수점 표시

```sql
SELECT MAX(LENGTH)
FROM FISH_INFO;
```

`MAX()`는 값을 계산해서 숫자 형태로 반환한다.

숫자 값은 출력 환경에 따라 뒤의 불필요한 소수점이 생략될 수 있다.

예를 들어 가장 큰 길이가 `39.00`이어도 단순히 `MAX(LENGTH)`만 조회하면 결과가 아래처럼 보일 수 있다.

| MAX(LENGTH) |
| ----------- |
| 39          |

문제에서 `39.00cm`처럼 소수점 둘째 자리까지 붙인 문자열을 요구한다면, 단순 `MAX()`만으로는 출력 형식을 맞추기 어렵다.

#### 3. FORMAT 함수

```sql
FORMAT(MAX(LENGTH), 2)
```

`FORMAT(값, 소수점 자리수)`는 숫자를 지정한 소수점 자리수까지 표시해 문자열 형태로 반환한다.

```sql
FORMAT(39, 2)
```

위 결과는 다음과 같다.

| 결과 |
| ---- |
| 39.00 |

따라서 `FORMAT(MAX(LENGTH), 2)`를 사용하면 가장 큰 길이를 소수점 둘째 자리까지 표시할 수 있다.

#### 4. CONCAT 함수

```sql
CONCAT(FORMAT(MAX(LENGTH), 2), 'cm')
```

`CONCAT()`은 여러 값을 하나의 문자열로 이어 붙일 때 사용한다.

이 문제에서는 숫자 형식으로 포매팅한 길이 뒤에 단위인 `cm`를 붙여야 한다.

```sql
CONCAT('39.00', 'cm')
```

위 결과는 다음과 같다.

| 결과 |
| ---- |
| 39.00cm |

#### 5. AS로 컬럼명 지정

```sql
AS MAX_LENGTH
```

`AS`는 조회 결과의 컬럼 이름을 별칭으로 지정할 때 사용한다.

문제에서 결과 컬럼명을 `MAX_LENGTH`로 요구하므로 아래처럼 별칭을 붙인다.

```sql
SELECT CONCAT(FORMAT(MAX(LENGTH), 2), 'cm') AS MAX_LENGTH
```

### 추가로 알아둘 내용

#### 1. FORMAT은 숫자를 문자열로 바꾼다

`FORMAT()`은 숫자를 보기 좋은 형태로 포매팅하지만, 결과는 문자열이다.

```sql
SELECT FORMAT(1234.5, 2);
```

결과는 다음과 같다.

| FORMAT(1234.5, 2) |
| ----------------- |
| 1,234.50          |

천 단위 구분 쉼표가 들어가고, 소수점 둘째 자리까지 표시된다.

이 문제에서는 물고기 길이의 최댓값을 출력 형식에 맞춰 보여주는 것이 목적이므로 `FORMAT()`을 사용해도 적절하다.

#### 2. CONCAT을 사용하지 않으면 단위를 붙일 수 없다

아래 SQL은 가장 큰 길이를 소수점 둘째 자리까지 표시할 수는 있지만, `cm` 단위가 붙지 않는다.

```sql
SELECT FORMAT(MAX(LENGTH), 2) AS MAX_LENGTH
FROM FISH_INFO;
```

결과는 다음처럼 나온다.

| MAX_LENGTH |
| ---------- |
| 39.00      |

문제에서 `39.00cm`처럼 단위까지 포함한 값을 요구한다면 `CONCAT()`으로 문자열을 이어 붙여야 한다.

```sql
SELECT CONCAT(FORMAT(MAX(LENGTH), 2), 'cm') AS MAX_LENGTH
FROM FISH_INFO;
```

#### 3. 실행 흐름으로 이해하기

정답 SQL은 안쪽 함수부터 차례대로 생각하면 이해하기 쉽다.

```sql
CONCAT(FORMAT(MAX(LENGTH), 2), 'cm')
```

먼저 `MAX(LENGTH)`가 가장 큰 길이를 구한다.

그다음 `FORMAT(..., 2)`가 값을 소수점 둘째 자리까지 표시한다.

마지막으로 `CONCAT(..., 'cm')`가 뒤에 `cm`를 붙인다.

예를 들어 가장 큰 길이가 `39`라면 아래 순서로 처리된다.

```text
MAX(LENGTH)                  -> 39
FORMAT(MAX(LENGTH), 2)       -> '39.00'
CONCAT(FORMAT(...), 'cm')    -> '39.00cm'
```

#### 4. ROUND와 FORMAT의 차이

`ROUND()`도 소수점 자리수를 조절할 수 있지만, 숫자 값을 반올림해서 반환한다.

```sql
SELECT ROUND(MAX(LENGTH), 2)
FROM FISH_INFO;
```

`ROUND()`는 계산용 숫자를 다룰 때 적합하다.

반면 `FORMAT()`은 소수점 자리수와 출력 모양을 맞춘 문자열을 만들 때 적합하다.

이 문제는 계산 결과를 다시 계산에 사용하는 것이 아니라, `cm`까지 붙여 출력 형식을 맞추는 문제이므로 `FORMAT()`과 `CONCAT()`을 함께 사용하는 방식이 잘 맞는다.
