# SQL 문법 정리

## 가장 큰 물고기 10마리 구하기

### 사용한 SQL

```sql
SELECT ID, LENGTH
FROM FISH_INFO
WHERE LENGTH IS NOT NULL
ORDER BY LENGTH DESC, ID ASC
LIMIT 10;
```

### 핵심 문법

#### 1. 특정 컬럼 조회

```sql
SELECT ID, LENGTH
```

필요한 컬럼만 선택해서 조회할 수 있다.

이 문제에서는 물고기의 `ID`와 `LENGTH`만 출력하면 되므로 두 컬럼만 선택했다.

#### 2. NULL 제외

```sql
WHERE LENGTH IS NOT NULL
```

`NULL`은 값이 없음을 의미한다.

`IS NOT NULL`은 해당 컬럼에 값이 존재하는 행만 조회할 때 사용한다.

이 문제에서는 길이 정보가 없는 물고기를 제외하기 위해 사용했다.

#### 3. ORDER BY 다중 정렬

```sql
ORDER BY LENGTH DESC, ID ASC
```

`ORDER BY`는 결과를 정렬할 때 사용한다.

- `LENGTH DESC`: 길이가 긴 물고기부터 내림차순 정렬
- `ID ASC`: 길이가 같다면 ID가 작은 물고기부터 오름차순 정렬

정렬 조건을 여러 개 사용할 때는 앞에 있는 조건이 먼저 적용되고, 값이 같을 때 다음 조건이 적용된다.

#### 4. LIMIT

```sql
LIMIT 10
```

`LIMIT`은 조회 결과에서 가져올 행의 개수를 제한한다.

이 문제에서는 가장 큰 물고기 10마리만 출력해야 하므로 `LIMIT 10`을 사용했다.

### 추가로 공부할 내용

#### 1. NULL을 제외하지 않으면 생길 수 있는 문제

이 문제에서는 길이 정보가 없는 물고기를 제외해야 하므로 아래 조건이 필요하다.

```sql
WHERE LENGTH IS NOT NULL
```

`NULL`은 숫자 `0`과 다르다.

`0`은 실제 값이 0인 것이고, `NULL`은 값 자체가 없다는 뜻이다.

그래서 `LENGTH != NULL`처럼 비교 연산자를 사용하면 원하는 결과가 나오지 않는다.

```sql
-- 잘못된 방식
WHERE LENGTH != NULL
```

`NULL` 여부를 검사할 때는 반드시 `IS NULL` 또는 `IS NOT NULL`을 사용해야 한다.

#### 2. 정렬 조건의 우선순위

```sql
ORDER BY LENGTH DESC, ID ASC
```

정렬 조건이 여러 개일 때는 앞 조건부터 차례대로 적용된다.

먼저 `LENGTH DESC`로 길이가 긴 순서대로 정렬하고, 길이가 같은 물고기가 있으면 `ID ASC`로 ID가 작은 순서대로 정렬한다.

예를 들어 아래와 같은 데이터가 있다고 하자.

| ID | LENGTH |
| -- | ------ |
| 3  | 50     |
| 1  | 50     |
| 2  | 70     |

정렬 결과는 다음과 같다.

| ID | LENGTH |
| -- | ------ |
| 2  | 70     |
| 1  | 50     |
| 3  | 50     |

길이 50인 물고기끼리는 `ID ASC` 조건에 의해 `1`, `3` 순서가 된다.

#### 3. LIMIT은 정렬 후에 적용된다

```sql
ORDER BY LENGTH DESC, ID ASC
LIMIT 10
```

`LIMIT 10`은 아무 10개를 가져오는 것이 아니라, 정렬이 끝난 결과에서 앞의 10개를 가져온다.

따라서 상위 N개를 구하는 문제에서는 보통 `ORDER BY`와 `LIMIT`을 같이 사용한다.

#### 4. 윈도우 함수로 푸는 방법

상위 10개를 구할 때 윈도우 함수를 사용할 수도 있다.

```sql
SELECT ID, LENGTH
FROM (
    SELECT
        ID,
        LENGTH,
        ROW_NUMBER() OVER (ORDER BY LENGTH DESC, ID ASC) AS RN
    FROM FISH_INFO
    WHERE LENGTH IS NOT NULL
) RANKED_FISH
WHERE RN <= 10;
```

`ROW_NUMBER()`는 정렬 기준에 따라 각 행에 1부터 번호를 붙인다.

그다음 바깥 쿼리에서 `RN <= 10` 조건으로 상위 10개만 가져온다.

이 문제에서는 `ORDER BY ... LIMIT 10`이 더 간단하지만, 윈도우 함수는 "그룹별 상위 N개" 문제를 풀 때 특히 많이 사용된다.

#### 5. LIMIT과 동점 처리

현재 풀이의 `LIMIT 10`은 정렬된 결과에서 정확히 10개만 가져온다.

만약 10번째 물고기와 같은 길이를 가진 물고기가 더 있어도, `LIMIT 10`은 10개까지만 자른다.

동점까지 모두 포함해야 하는 문제라면 `RANK()`를 사용할 수 있다.

```sql
SELECT ID, LENGTH
FROM (
    SELECT
        ID,
        LENGTH,
        RANK() OVER (ORDER BY LENGTH DESC) AS LENGTH_RANK
    FROM FISH_INFO
    WHERE LENGTH IS NOT NULL
) RANKED_FISH
WHERE LENGTH_RANK <= 10
ORDER BY LENGTH DESC, ID ASC;
```

`RANK()`는 같은 길이에 같은 순위를 부여한다.

다만 이 문제는 "10마리"를 요구하므로 정확히 10개를 가져오는 `LIMIT 10` 풀이가 적절하다.
