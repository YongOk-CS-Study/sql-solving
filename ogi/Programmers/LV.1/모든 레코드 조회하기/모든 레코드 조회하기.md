# SQL 문법 정리

## 모든 레코드 조회하기

### 사용한 SQL

```sql
SELECT *
FROM ANIMAL_INS
ORDER BY ANIMAL_ID;
```

### 핵심 문법

#### 1. 모든 컬럼 조회

```sql
SELECT *
```

`SELECT`는 조회할 컬럼을 선택할 때 사용한다.

`*`는 모든 컬럼을 조회한다는 의미이다.

이 문제에서는 `ANIMAL_INS` 테이블의 모든 정보를 출력해야 하므로 특정 컬럼명을 하나씩 적지 않고 `*`를 사용한다.

#### 2. FROM

```sql
FROM ANIMAL_INS
```

`FROM`은 데이터를 조회할 테이블을 지정할 때 사용한다.

이 문제에서는 동물 보호소에 들어온 동물의 정보가 들어 있는 `ANIMAL_INS` 테이블에서 데이터를 조회한다.

#### 3. ORDER BY 오름차순 정렬

```sql
ORDER BY ANIMAL_ID
```

`ORDER BY`는 조회 결과를 특정 컬럼 기준으로 정렬할 때 사용한다.

정렬 방향을 따로 작성하지 않으면 기본값은 오름차순이다.

따라서 아래 두 SQL은 같은 의미이다.

```sql
ORDER BY ANIMAL_ID;
```

```sql
ORDER BY ANIMAL_ID ASC;
```

이 문제에서는 `ANIMAL_ID` 순서대로 결과를 출력해야 하므로 `ORDER BY ANIMAL_ID`를 사용한다.

### 추가로 알아둘 내용

#### 1. SELECT *의 의미

```sql
SELECT *
FROM ANIMAL_INS;
```

`SELECT *`는 테이블에 있는 모든 컬럼을 조회한다.

`ANIMAL_INS` 테이블에 아래와 같은 컬럼이 있다면 모두 출력된다.

| 컬럼명 | 의미 |
| ------ | ---- |
| ANIMAL_ID | 동물의 아이디 |
| ANIMAL_TYPE | 동물의 종류 |
| DATETIME | 보호 시작일 |
| INTAKE_CONDITION | 보호 시작 시 상태 |
| NAME | 이름 |
| SEX_UPON_INTAKE | 성별 및 중성화 여부 |

#### 2. ORDER BY가 필요한 이유

SQL에서 정렬 조건을 지정하지 않으면 조회 결과의 순서가 항상 보장되지 않는다.

문제에서 특정 순서로 출력하라고 요구한다면 반드시 `ORDER BY`를 사용해야 한다.

```sql
SELECT *
FROM ANIMAL_INS
ORDER BY ANIMAL_ID;
```

위 SQL은 모든 동물 정보를 조회한 뒤, `ANIMAL_ID`가 작은 값부터 큰 값 순서로 정렬한다.

#### 3. ASC와 DESC

`ASC`는 오름차순 정렬을 의미한다.

```sql
ORDER BY ANIMAL_ID ASC
```

`DESC`는 내림차순 정렬을 의미한다.

```sql
ORDER BY ANIMAL_ID DESC
```

이 문제는 아이디 순으로 조회해야 하므로 오름차순인 `ASC`를 사용하면 된다.

다만 `ASC`는 기본값이므로 생략할 수 있다.
