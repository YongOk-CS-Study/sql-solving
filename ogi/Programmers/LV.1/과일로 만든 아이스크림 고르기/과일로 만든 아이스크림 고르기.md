# SQL 문법 정리

## 과일로 만든 아이스크림 고르기

### 사용한 SQL

```sql
SELECT i.FLAVOR
FROM ICECREAM_INFO AS i
JOIN FIRST_HALF AS fh
    ON i.FLAVOR = fh.FLAVOR
WHERE fh.TOTAL_ORDER > 3000
    AND i.INGREDIENT_TYPE = 'fruit_based'
ORDER BY fh.TOTAL_ORDER DESC;
```

### 핵심 문법

#### 1. 테이블 별칭

```sql
FROM ICECREAM_INFO AS i
JOIN FIRST_HALF AS fh
```

`AS`는 테이블이나 컬럼에 별칭을 붙일 때 사용한다.

`ICECREAM_INFO`를 `i`, `FIRST_HALF`를 `fh`라는 짧은 이름으로 사용할 수 있다.

이렇게 별칭을 붙이면 컬럼을 작성할 때 어느 테이블의 컬럼인지 명확하게 구분할 수 있다.

```sql
i.FLAVOR
fh.TOTAL_ORDER
```

#### 2. JOIN

```sql
JOIN FIRST_HALF AS fh
    ON i.FLAVOR = fh.FLAVOR
```

`JOIN`은 두 테이블의 데이터를 연결해서 조회할 때 사용한다.

이 문제에서는 아이스크림의 성분 정보가 있는 `ICECREAM_INFO` 테이블과 상반기 주문 정보가 있는 `FIRST_HALF` 테이블을 함께 사용해야 한다.

두 테이블은 `FLAVOR` 컬럼을 기준으로 연결한다.

```sql
ON i.FLAVOR = fh.FLAVOR
```

즉, 같은 맛을 가진 행끼리 연결해서 하나의 결과로 조회한다.

#### 3. WHERE 조건

```sql
WHERE fh.TOTAL_ORDER > 3000
    AND i.INGREDIENT_TYPE = 'fruit_based'
```

`WHERE`는 원하는 조건에 맞는 행만 조회할 때 사용한다.

이 문제에서는 두 가지 조건을 모두 만족해야 한다.

- 총주문량이 3,000보다 커야 한다.
- 주 성분이 과일이어야 한다.

두 조건을 모두 만족해야 하므로 `AND`를 사용한다.

#### 4. 문자열 비교

```sql
i.INGREDIENT_TYPE = 'fruit_based'
```

SQL에서 문자열 값은 작은따옴표로 감싸서 작성한다.

`fruit_based`는 컬럼명이 아니라 실제 데이터 값이다.

따라서 아래처럼 작성해야 한다.

```sql
WHERE i.INGREDIENT_TYPE = 'fruit_based'
```

만약 작은따옴표 없이 작성하면 SQL은 `fruit_based`를 문자열 값이 아니라 컬럼명이나 식별자로 해석하려고 한다.

```sql
WHERE i.INGREDIENT_TYPE = fruit_based
```

위처럼 작성하면 `fruit_based`라는 컬럼을 찾으려고 하기 때문에 오류가 발생할 수 있다.

#### 5. ORDER BY 내림차순 정렬

```sql
ORDER BY fh.TOTAL_ORDER DESC
```

`ORDER BY`는 조회 결과를 특정 컬럼 기준으로 정렬할 때 사용한다.

`DESC`는 내림차순 정렬을 의미한다.

이 문제에서는 총주문량이 큰 순서대로 아이스크림 맛을 출력해야 하므로 `fh.TOTAL_ORDER DESC`를 사용한다.

### 추가로 알아둘 내용

#### 1. 숫자와 문자열 조건의 차이

숫자 값은 작은따옴표 없이 비교한다.

```sql
fh.TOTAL_ORDER > 3000
```

문자열 값은 작은따옴표로 감싸서 비교한다.

```sql
i.INGREDIENT_TYPE = 'fruit_based'
```

`3000`은 숫자이므로 그대로 작성하고, `fruit_based`는 문자열이므로 `'fruit_based'`처럼 작성한다.

#### 2. 작은따옴표와 큰따옴표

SQL에서 문자열은 보통 작은따옴표를 사용한다.

```sql
'fruit_based'
```

데이터베이스 종류나 설정에 따라 큰따옴표의 의미가 달라질 수 있으므로, 문자열 값은 작은따옴표로 쓰는 습관을 들이는 것이 좋다.

#### 3. SELECT에는 출력할 컬럼만 작성한다

문제에서 출력해야 하는 값은 아이스크림 맛이다.

따라서 `SELECT`에는 `i.FLAVOR`만 작성한다.

```sql
SELECT i.FLAVOR
```

조건에 사용하는 `fh.TOTAL_ORDER`, `i.INGREDIENT_TYPE`은 출력할 필요가 없으므로 `SELECT`에 포함하지 않아도 된다.

#### 4. 쿼리 실행 흐름으로 이해하기

아래 순서로 생각하면 SQL을 이해하기 쉽다.

```text
1. ICECREAM_INFO와 FIRST_HALF를 FLAVOR 기준으로 연결한다.
2. TOTAL_ORDER가 3000보다 큰 행만 남긴다.
3. INGREDIENT_TYPE이 'fruit_based'인 행만 남긴다.
4. TOTAL_ORDER가 큰 순서대로 정렬한다.
5. FLAVOR만 출력한다.
```

실제 SQL 작성 순서와 내부 실행 순서는 완전히 같지 않을 수 있지만, 문제를 풀 때는 위 흐름으로 이해하면 조건을 놓치지 않기 좋다.
