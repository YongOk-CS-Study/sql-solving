# SQL 문법 정리

## 이름이 없는 동물의 아이디

### 사용한 SQL

```sql
SELECT ANIMAL_ID
FROM ANIMAL_INS
WHERE NAME IS NULL
ORDER BY ANIMAL_ID;
```

### 문제 이해하기

이 문제는 동물 보호소에 들어온 동물의 정보를 담은 `ANIMAL_INS` 테이블에서 이름이 없는 동물의 아이디를 조회하는 문제입니다.

이름 정보는 `NAME` 컬럼에 저장되어 있습니다.

이름이 없는 동물은 `NAME` 값이 비어 있는 상태이므로, SQL에서는 `NAME IS NULL` 조건으로 찾을 수 있습니다.

출력해야 하는 값은 동물의 아이디인 `ANIMAL_ID`이고, 결과는 `ANIMAL_ID` 순으로 정렬해야 합니다.

### 핵심 문법

#### 1. SELECT

```sql
SELECT ANIMAL_ID
```

`SELECT`는 조회할 컬럼을 선택할 때 사용합니다.

이 문제에서는 이름이 없는 동물의 아이디만 출력하면 되므로 `ANIMAL_ID` 컬럼만 선택합니다.

#### 2. FROM

```sql
FROM ANIMAL_INS
```

`FROM`은 데이터를 조회할 테이블을 지정할 때 사용합니다.

동물 보호소에 들어온 동물의 정보는 `ANIMAL_INS` 테이블에 들어 있으므로 해당 테이블에서 데이터를 조회합니다.

#### 3. WHERE

```sql
WHERE NAME IS NULL
```

`WHERE`는 원하는 조건에 맞는 행만 조회할 때 사용합니다.

이 문제에서는 이름이 없는 동물만 찾아야 하므로 `NAME` 컬럼이 `NULL`인 행만 조회합니다.

#### 4. IS NULL

```sql
NAME IS NULL
```

SQL에서 `NULL`은 값이 없음을 의미합니다.

`NULL`은 일반적인 값처럼 `=` 연산자로 비교할 수 없습니다.

따라서 아래와 같이 작성하면 올바른 조건식이 아닙니다.

```sql
WHERE NAME = NULL
```

`NULL` 여부를 확인할 때는 반드시 `IS NULL`을 사용해야 합니다.

```sql
WHERE NAME IS NULL
```

반대로 값이 비어 있지 않은 데이터를 찾고 싶다면 `IS NOT NULL`을 사용할 수 있습니다.

```sql
WHERE NAME IS NOT NULL
```

#### 5. ORDER BY

```sql
ORDER BY ANIMAL_ID
```

`ORDER BY`는 조회 결과를 특정 컬럼 기준으로 정렬할 때 사용합니다.

정렬 방향을 따로 적지 않으면 기본값은 오름차순입니다.

따라서 아래 두 문장은 같은 의미입니다.

```sql
ORDER BY ANIMAL_ID;
```

```sql
ORDER BY ANIMAL_ID ASC;
```

이 문제에서는 아이디 순으로 조회해야 하므로 `ORDER BY ANIMAL_ID`를 사용합니다.

### 실행 흐름

1. `ANIMAL_INS` 테이블에서 데이터를 조회합니다.
2. `NAME IS NULL` 조건으로 이름이 없는 동물만 필터링합니다.
3. 필터링된 결과에서 `ANIMAL_ID`만 출력합니다.
4. `ANIMAL_ID` 기준으로 오름차순 정렬합니다.

### 최종 정리

이 문제의 핵심은 이름이 없는 데이터를 찾기 위해 `NULL` 조건을 올바르게 사용하는 것입니다.

`NULL`은 `= NULL`로 비교하지 않고, `IS NULL`로 확인해야 합니다.

```sql
SELECT ANIMAL_ID
FROM ANIMAL_INS
WHERE NAME IS NULL
ORDER BY ANIMAL_ID;
```
