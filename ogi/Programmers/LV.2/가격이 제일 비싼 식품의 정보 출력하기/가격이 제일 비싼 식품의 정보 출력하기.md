# SQL 문법 정리

## 가격이 제일 비싼 식품의 정보 출력하기

### 사용한 SQL

```sql
SELECT *
FROM FOOD_PRODUCT
WHERE PRICE = (
    SELECT MAX(PRICE)
    FROM FOOD_PRODUCT
);
```

### 핵심 문법

#### 1. SELECT *

```sql
SELECT *
```

`*`는 테이블의 모든 컬럼을 조회한다는 의미다.

이 문제에서는 가격이 가장 비싼 식품의 전체 정보를 출력해야 하므로 `SELECT *`를 사용했다.

#### 2. WHERE 조건

```sql
WHERE PRICE = (...)
```

`WHERE`는 조회할 행을 조건으로 필터링할 때 사용한다.

여기서는 `PRICE`가 가장 비싼 가격과 같은 상품만 조회한다.

#### 3. MAX 함수

```sql
SELECT MAX(PRICE)
FROM FOOD_PRODUCT
```

`MAX()`는 특정 컬럼에서 가장 큰 값을 구하는 집계 함수다.

이 문제에서는 `PRICE` 컬럼의 최댓값, 즉 가장 비싼 가격을 구하는 데 사용했다.

#### 4. 서브쿼리

```sql
WHERE PRICE = (
    SELECT MAX(PRICE)
    FROM FOOD_PRODUCT
)
```

서브쿼리는 SQL문 안에 들어가는 또 다른 SQL문이다.

먼저 서브쿼리에서 가장 비싼 가격을 구하고, 바깥 쿼리에서 해당 가격을 가진 식품 정보를 조회한다.

#### 5. ORDER BY와 LIMIT을 이용한 다른 풀이

```sql
SELECT *
FROM FOOD_PRODUCT
ORDER BY PRICE DESC
LIMIT 1;
```

`ORDER BY PRICE DESC`는 가격을 내림차순으로 정렬한다.

`LIMIT 1`은 정렬된 결과 중 가장 위의 1개 행만 가져온다.

가격이 가장 큰 상품이 하나만 나온다는 조건에서는 이 방식으로도 가장 비싼 식품을 구할 수 있다.

### 추가로 공부할 내용

#### 1. 서브쿼리 풀이와 LIMIT 풀이의 차이

이 문제는 아래 두 방식으로 풀 수 있다.

```sql
SELECT *
FROM FOOD_PRODUCT
WHERE PRICE = (
    SELECT MAX(PRICE)
    FROM FOOD_PRODUCT
);
```

```sql
SELECT *
FROM FOOD_PRODUCT
ORDER BY PRICE DESC
LIMIT 1;
```

두 풀이 모두 가장 비싼 상품을 찾는다는 점은 같지만, 결과가 달라질 수 있는 상황이 있다.

만약 가장 비싼 가격을 가진 상품이 여러 개라면 `WHERE PRICE = MAX(PRICE)` 방식은 해당 상품들을 모두 가져온다.

반면 `ORDER BY PRICE DESC LIMIT 1` 방식은 정렬된 결과 중 1개만 가져온다.

따라서 "최댓값에 해당하는 모든 행"을 가져와야 한다면 서브쿼리 방식이 더 정확하다.

#### 2. 집계 함수는 WHERE에서 바로 사용할 수 없다

아래처럼 `WHERE` 조건에 `MAX(PRICE)`를 바로 쓰면 일반적으로 올바른 문법이 아니다.

```sql
SELECT *
FROM FOOD_PRODUCT
WHERE PRICE = MAX(PRICE);
```

`WHERE`는 행을 하나씩 검사하는 단계이고, `MAX()`는 여러 행을 모아서 계산하는 집계 함수다.

그래서 먼저 서브쿼리에서 `MAX(PRICE)`를 계산한 뒤, 바깥 쿼리의 `WHERE`에서 그 값을 비교한다.

#### 3. 실행 순서 관점으로 보기

SQL은 작성 순서와 실제 처리 순서가 다르다.

이 문제를 이해할 때는 대략 아래 순서로 생각하면 좋다.

```text
FROM -> WHERE -> SELECT -> ORDER BY -> LIMIT
```

서브쿼리 풀이에서는 안쪽 쿼리가 먼저 `MAX(PRICE)`를 구하고, 바깥 쿼리가 그 값을 이용해 `WHERE` 조건을 검사한다.

#### 4. 동점 상품까지 고려하는 윈도우 함수 풀이

윈도우 함수를 사용하면 가격 순위를 매긴 뒤 1등 상품을 가져올 수도 있다.

```sql
SELECT PRODUCT_ID, PRODUCT_NAME, PRODUCT_CD, CATEGORY, PRICE
FROM (
    SELECT
        FOOD_PRODUCT.*,
        RANK() OVER (ORDER BY PRICE DESC) AS PRICE_RANK
    FROM FOOD_PRODUCT
) RANKED_PRODUCT
WHERE PRICE_RANK = 1;
```

`RANK()`는 같은 가격이면 같은 순위를 부여한다.

따라서 가장 비싼 가격의 상품이 여러 개라면 모두 `PRICE_RANK = 1`이 된다.

다만 이 문제에서는 `MAX()` 서브쿼리 풀이가 더 간단하고 적합하다.

#### 5. SELECT * 사용 시 주의점

문제에서 모든 컬럼을 요구하면 `SELECT *`를 사용해도 된다.

하지만 실무나 복잡한 쿼리에서는 필요한 컬럼을 명시하는 편이 좋다.

```sql
SELECT PRODUCT_ID, PRODUCT_NAME, PRODUCT_CD, CATEGORY, PRICE
FROM FOOD_PRODUCT
WHERE PRICE = (
    SELECT MAX(PRICE)
    FROM FOOD_PRODUCT
);
```

컬럼을 직접 적으면 결과 구조를 명확하게 알 수 있고, 불필요한 데이터를 조회하지 않을 수 있다.
