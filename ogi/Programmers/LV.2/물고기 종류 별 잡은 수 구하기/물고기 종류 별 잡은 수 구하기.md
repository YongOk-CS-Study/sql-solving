# SQL 문법 정리

## 물고기 종류 별 잡은 수 구하기

### 사용한 SQL

```sql
SELECT
    COUNT(ID) AS FISH_COUNT,
    NAME.FISH_NAME
FROM FISH_INFO AS INFO
JOIN FISH_NAME_INFO AS NAME
    ON NAME.FISH_TYPE = INFO.FISH_TYPE
GROUP BY NAME.FISH_TYPE, NAME.FISH_NAME
ORDER BY FISH_COUNT DESC;
```

### 문제 이해하기

이 문제는 잡은 물고기 정보가 들어 있는 `FISH_INFO` 테이블과 물고기 이름 정보가 들어 있는 `FISH_NAME_INFO` 테이블을 사용해서, 물고기 이름별로 몇 마리를 잡았는지 구하는 문제입니다.

`FISH_INFO` 테이블에는 물고기의 종류가 숫자인 `FISH_TYPE`으로 저장되어 있습니다.

`FISH_NAME_INFO` 테이블에는 같은 `FISH_TYPE`에 해당하는 실제 물고기 이름인 `FISH_NAME`이 저장되어 있습니다.

따라서 두 테이블을 `FISH_TYPE` 기준으로 연결한 뒤, 물고기 종류와 이름별로 묶어서 개수를 세면 됩니다.

출력 컬럼명은 문제에서 요구한 대로 잡은 수는 `FISH_COUNT`, 물고기 이름은 `FISH_NAME`으로 지정합니다.

### 핵심 문법

#### 1. JOIN

```sql
JOIN FISH_NAME_INFO AS NAME
    ON NAME.FISH_TYPE = INFO.FISH_TYPE
```

`JOIN`은 두 테이블의 데이터를 연결해서 조회할 때 사용합니다.

이 문제에서는 `FISH_INFO` 테이블의 `FISH_TYPE`과 `FISH_NAME_INFO` 테이블의 `FISH_TYPE`이 같은 행을 연결합니다.

이렇게 연결하면 숫자로 저장된 물고기 종류에 해당하는 물고기 이름을 함께 조회할 수 있습니다.

#### 2. COUNT()

```sql
COUNT(ID) AS FISH_COUNT
```

`COUNT()`는 행의 개수를 세는 집계 함수입니다.

이 문제에서는 물고기 종류별로 잡은 수를 구해야 하므로 `COUNT(ID)`를 사용합니다.

`ID`는 `NULL`이 될 수 없는 컬럼이기 때문에, 잡힌 물고기 한 행을 하나씩 세는 용도로 사용할 수 있습니다.

#### 3. GROUP BY

```sql
GROUP BY NAME.FISH_TYPE, NAME.FISH_NAME
```

`GROUP BY`는 같은 값을 가진 행들을 하나의 그룹으로 묶을 때 사용합니다.

이 문제에서는 물고기 종류별 잡은 수를 구해야 하므로 `FISH_TYPE`을 기준으로 그룹화합니다.

또한 결과에 `FISH_NAME`을 출력해야 하므로 `FISH_NAME`도 함께 `GROUP BY`에 작성합니다.

#### 4. ORDER BY

```sql
ORDER BY FISH_COUNT DESC
```

`ORDER BY`는 조회 결과를 정렬할 때 사용합니다.

문제에서 잡은 수 기준으로 내림차순 정렬하라고 했으므로, `FISH_COUNT DESC`를 사용합니다.

`DESC`는 큰 값부터 작은 값 순서로 정렬한다는 뜻입니다.

### 풀이 흐름

1. `FISH_INFO` 테이블을 기준으로 데이터를 조회합니다.
2. `FISH_NAME_INFO` 테이블을 `FISH_TYPE` 기준으로 조인합니다.
3. 물고기 종류와 이름별로 그룹화합니다.
4. `COUNT(ID)`로 각 그룹의 물고기 수를 셉니다.
5. 잡은 수 컬럼명을 `FISH_COUNT`로 지정합니다.
6. `FISH_COUNT` 기준으로 내림차순 정렬합니다.

### 최종 정리

이 문제의 핵심은 물고기 종류 번호인 `FISH_TYPE`을 이용해 두 테이블을 조인하는 것입니다.

그다음 `GROUP BY`로 물고기 종류별 그룹을 만들고, `COUNT()`로 각 그룹에 속한 행의 개수를 세면 잡은 수를 구할 수 있습니다.

마지막으로 `ORDER BY FISH_COUNT DESC`를 사용해 많이 잡은 물고기부터 출력하면 됩니다.

### 추가로 공부할 내용

#### 1. COUNT(*)를 사용해도 되는 이유

이 문제는 물고기 한 마리가 `FISH_INFO` 테이블의 한 행으로 저장되어 있습니다.

따라서 아래처럼 `COUNT(*)`를 사용해도 같은 결과를 얻을 수 있습니다.

```sql
SELECT
    COUNT(*) AS FISH_COUNT,
    NAME.FISH_NAME
FROM FISH_INFO AS INFO
JOIN FISH_NAME_INFO AS NAME
    ON NAME.FISH_TYPE = INFO.FISH_TYPE
GROUP BY NAME.FISH_TYPE, NAME.FISH_NAME
ORDER BY FISH_COUNT DESC;
```

`COUNT(*)`는 그룹에 속한 전체 행의 개수를 셉니다.

반면 `COUNT(ID)`는 `ID`가 `NULL`이 아닌 행의 개수를 셉니다.

이 문제에서 `ID`는 `NULL`이 될 수 없으므로 두 방식의 결과가 같습니다.

#### 2. GROUP BY에 FISH_NAME을 함께 쓰는 이유

아래처럼 `FISH_TYPE`만 그룹화하고 `FISH_NAME`을 조회하면 SQL 실행 환경에 따라 오류가 발생할 수 있습니다.

```sql
GROUP BY NAME.FISH_TYPE
```

집계 함수로 계산하지 않은 컬럼을 `SELECT`에서 조회하려면, 일반적으로 해당 컬럼도 `GROUP BY`에 포함해야 합니다.

그래서 다음처럼 작성하는 것이 더 명확하고 안전합니다.

```sql
GROUP BY NAME.FISH_TYPE, NAME.FISH_NAME
```

#### 3. 테이블 별칭을 사용하는 이유

```sql
FROM FISH_INFO AS INFO
JOIN FISH_NAME_INFO AS NAME
```

테이블 별칭을 사용하면 긴 테이블명을 짧게 줄여서 쓸 수 있습니다.

또한 두 테이블에 같은 이름의 컬럼인 `FISH_TYPE`이 있으므로, `INFO.FISH_TYPE`, `NAME.FISH_TYPE`처럼 어느 테이블의 컬럼인지 명확하게 표현할 수 있습니다.
